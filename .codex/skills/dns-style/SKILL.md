# DNS Visual Style Skill

## Purpose

Use this skill when a task asks for a DNS-like, corporate, retail, dashboard, simulation, or management-interface visual style.

The style must support clarity and business use. It must not become decorative noise.

## Visual Direction

Prefer:
- strict business layout;
- clear card hierarchy;
- orange as the main corporate accent;
- graphite, black, white, and light gray as the base;
- blue only as an analytical accent;
- red only for real risks or critical deviations;
- readable FullHD screens;
- practical interface patterns close to working tools, not fantasy game art.

Avoid:
- generic AI gradients;
- excessive glassmorphism;
- low-contrast text;
- overloaded dashboards;
- decorative backgrounds that fight with content;
- meaningless neon effects;
- visual ideas that hide product logic.

## Layout Principles

Each screen should show:
- what is happening now;
- what changed;
- why it matters;
- what the user can do next.

For dashboards and management screens, prefer the chain:

`indicator -> deviation -> reason -> action`

For simulation screens, preserve:
- scenario logic;
- available actions;
- resource/state visibility;
- feedback after decisions;
- evaluator/admin controls.

## Copy Rules

Use short, human labels.
Avoid formal filler and vague motivational text.
A label must help the user act or understand state.

## Verification

After applying DNS style, check:
- content remains readable;
- controls remain visible;
- visual emphasis matches business importance;
- colors do not create false urgency;
- the screen still works without relying on decoration.
