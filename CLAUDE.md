## Коннектор к Obsidian (база знаний Claude)

Знания живут в **одном месте** — Obsidian-vault `Pedro78`, папка `claude-kb/`.

- **MCP-сервер:** `obsidian`, префикс путей — `Pedro78/`.
- **Vault на диске:** `D:\MyProject\Obsidian\Pedro78`.
- **База знаний (единая точка):** `Pedro78/claude-kb/`.
- **Точка входа:** `Pedro78/claude-kb/INDEX.md` — **читать первой** при работе со знаниями.

### Что внутри `claude-kb/`

| Путь | Назначение |
|------|-----------|
| `claude-kb/INDEX.md` | Точка входа (коннектор). Карта базы + как ею пользоваться. |
| `claude-kb/raw/`  | **Сырые, непроверенные данные.** Выгрузки, прогоны, черновики, гипотезы, находки. Дёшево добавить, проверять необязательно. |
| `claude-kb/wiki/` | **Зафиксированный канон.** Только подтверждённое: решения, проверенные числа/формулы, утверждённая механика. Источник истины. |
| `claude-kb/CLAUDE.md` | Правила потока raw↔wiki (vault-сторона). |

### Как пользоваться
1. **Чтение:** открыть `claude-kb/INDEX.md`; источник истины — `wiki/`; `raw/` — справочно.
2. **Запись:** находка/прогон/идея → `raw/`; подтверждённое → промоут в `wiki/`.

### Поток данных

```
источник / прогон / идея  →  raw/  →  (проверка)  →  wiki/  →  репозиторий
```

- **код → Obsidian:** подтверждённые данные фиксировать в `wiki/`; сырое — в `raw/`.
- **Obsidian → код:** `wiki/` — источник истины; отражать в `docs/` и код.
- Канон по механике ЗРД дублируется в `docs/zrd-wiki/`. **При расхождении с vault `wiki/` — синхронизировать, не плодить две версии правды.**

### Правила
- Данные из `raw/` не истина, пока не проверены и не перенесены в `wiki/`.
- Промоушен raw → wiki только когда данные подтверждены (тест / решение / число зафиксировано).
- В `wiki/` соблюдать стандарт frontmatter vault и «≥1 wikilink».
- Удаление/переименование/перенос заметок в vault — только с подтверждения пользователя.

Подробные правила: `Pedro78/claude-kb/CLAUDE.md`. Конституция vault: `Pedro78/CLAUDE.md`.
