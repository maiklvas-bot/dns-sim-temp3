# Глоссарий DNS SimCenter

Два пласта терминов:
- **Термины ассессмента** — словарь самого продукта (см. раздел в конце).
- **Термины розницы DNS/РРС** — контекст, на котором строятся кейсы симуляции (директор управляет реальным магазином DNS).

**Источник терминов розницы:** WIKI_Глоссарий_РРС_Тюмень_DNS.docx (v1.0, 25.05.2026).

> Термины розницы нужны при создании и трактовке кейсов: ситуации симуляции происходят в реальном магазине DNS со складом, выдачей, сменой и клиентами.

# Термины ассессмента (SimCenter)

Словарь самого продукта. Источник истины — код (`shared/simulation-content.ts`, `shared/simulation-scoring.ts`, `script/bootstrap-content.json`).

## Оценка и компетенции

| Термин | Значение |
|--------|----------|
| **Ассессмент** | Оценка управленческих компетенций через симуляцию рабочего дня директора магазина |
| **Компетенция** | Измеряемый управленческий навык. Всего 14, шкала 0–5 |
| **Категория компетенции** | Группа: `basic` (базовые знания), `advanced` (продвинутые навыки), `leadership` (лидерство и управление) |
| **Целевой уровень** | Планка готовности — **4.0 из 5**; с ней сравнивается профиль |
| **Профиль компетенций** | Набор средних баллов по всем компетенциям участника |
| **Радар компетенций** | Лепестковая диаграмма профиля vs целевой линии 4.0 |
| **План развития** | Список компетенций ниже цели с рекомендациями |
| **Вердикт** | Итоговая оценка готовности (hero-карточка на странице результатов) |

## Контент симуляции

| Термин | Значение |
|--------|----------|
| **Кейс** (`SimCase`) | Управленческая ситуация. Имеет основные и второстепенные компетенции |
| **Цикл** (`CaseCycle`) | Шаг внутри кейса; кейс многошаговый, циклы могут ветвиться |
| **Сигнал** (`signal`) | Что видит/слышит участник в цикле: сообщение, сигнал зоны, письмо, звонок, визит |
| **Триггер** (`trigger`) | Как кейс приходит участнику (message / email / call / visitor / zone_signal) |
| **Вариант решения** (`option`) | Выбор участника. Имеет `level`, `score` (0–5), вклад в компетенции и эффекты |
| **Уровень варианта** (`level`) | Качество решения: 1 = слабое, выше = лучше |
| **Канал** | Источник сигнала: `main_case`, `email`, `messenger`, `video` |
| **Зона** | Часть магазина: торговый зал, склад, выдача, начальство |

## Расчёт оценки

| Термин | Значение |
|--------|----------|
| **score** | Балл варианта решения, 0–5 |
| **competency_scores** | Вклад варианта в конкретные компетенции |
| **effects** | Влияние решения на метрики магазина: очередь (queue), конверсия (conversion), настрой (morale), влияние на выручку (revenue_impact), статус выдачи (delivery_status) |
| **Вес кейса** (`caseWeight`) | Значимость кейса в %, нормируется к 0–1 |
| **Качество (qualityRatio)** | `score / 5`, ограничено 0.1–1 |
| **Сложность** | Уровень сессии: `easy` / `medium` / `hard` (разное число авто-кейсов и время) |
| **Временной коэффициент** | Множитель по сложности (easy 0.95 / medium 1.0 / hard 1.08), если включён |

## Процесс и роли

| Термин | Значение |
|--------|----------|
| **Live-сессия** | Прохождение симуляции участником под наблюдением Оценщика |
| **Администратор** (`admin`) | Роль: управление контентом, настройками, персоналом |
| **Оценщик** (`evaluator`) | Роль: запуск/мониторинг сессии (экран «Асессор») + просмотр результатов |
| **Асессор** | Не роль, а **экран запуска сессии**, которым пользуется Оценщик |
| **Участник** (`participant`) | Оцениваемый; входит по присоединению к сессии |
| **PDF-отчёт** | Экспорт результата с профилем, радаром и планом развития (с кириллицей) |
