# Обучающие материалы и ключевые сообщения

## Ключевое сообщение для всей команды
> «Дашборд нужен не для поиска виноватых. Нужен чтобы быстрее видеть проблему,
> понимать причину, назначать действие и проверять — стало ли лучше.»

## По ролям

| Роль | Сообщение |
|------|----------|
| УМ | Инструмент подготовки к управленческому разговору и контроля своего филиала |
| ЗУМ | Инструмент ежедневной операционной дисциплины |
| Аналитик | Единый контур данных и снижение ручной сборки |
| Дивизион | Управленческая практика РРС Тюмень — прозрачный контроль стратегии |

## Структура обучения

| Роль | Формат | Темы |
|------|--------|------|
| ДРРС | — (сам владелец) | — |
| УМ | Совещание ~30 мин | Экран → карточка → красные зоны → задачи → эффект |
| ЗУМ | Отдельная сессия | Операционный экран → поручения → постамат → ТСО → задачи дня |
| Аналитик | Технический onboarding | Источники → обновление → качество → выгрузка |
| Дивизион | Демо витрины | Агрегированный режим |

## Глоссарий в интерфейсе
Tooltip-подсказки к терминам: ТСО, ЧСО, СО, dok/час, МКФ, ВЕ, СЕ, КБТ, ТВ, ТП, ИЗ, ОСНТ, БА, ДА, ГУС, СТМ.
Пользователь не должен помнить всё наизусть.
