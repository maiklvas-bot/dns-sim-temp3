# Источники данных и частота обновления

## Внутренние системы DNS

| Источник | Что берём | Важно |
|---------|-----------|-------|
| **1С** | РН, КЗ, оплаты, графики, документы, Z-отчёты | Лаг обновления — показывать дату загрузки |
| **Web-база (ВБ)** | Заказы, события РН, выдача, PosCredit | WEB/ПК может подставлять среднее вместо факта |
| **МП** | Сканирование, поручения, инвентаризация | Точнее для процессов; зависит от устройства |
| **DNS Sale** | Поручения, конвейер, набор, выдача, роли | Ключевой для ЗУМ |
| **ClickHouse** | Приёмка, отгрузка, МКФ, расчёты, логи | При расхождении с 1С — показывать источник |
| **GreenPlum** | Логи МП, поручения, инвентаризация | Технический источник |
| **FRP** | Средние времена, процессные показатели, ранги | Непредставительно если документы вне временных границ |
| **Оперативный контроль** | Текущие продажи, пачки, дисциплина | Не единственный источник |

## Файловые и ручные источники

| Источник | Применение | Риск |
|---------|-----------|------|
| **Excel** | Итоги, рейтинги, планы, персонал | Разные версии, ручные корректировки |
| **Google Sheets** | Справочники, планы, задачи | Допустим для MVP |
| **Ручные справочники** | Список филиалов, МКФ, роли, ВЕ/СЕ | Критически важны — без них путаница с названиями |

> ⚠️ Справочник филиалов = критически важный элемент.

## Частота обновления

| Блок | Частота | Что |
|------|---------|-----|
| Оперативный | **Ежедневно** | Выручка, план-факт, dok/час, ТСО/ЧСО, постамат, красные зоны |
| Near real-time (след. уровень) | Несколько раз в день | Поручения, постамат, очередь |
| Регулярный | **Еженедельно** | Рейтинги, карточки, товарные разрывы, задачи |
| Финансовый | **Ежемесячно** | ОДР, рентабельность, ФОТ/об, персонал |

## Правило каждого показателя
Источник + Дата/время обновления + Метод расчёта + Владелец + Приоритет при расхождении
