# Барьеры внедрения

| Пользователь | Барьер | Ответ системы |
|-------------|--------|--------------|
| УМ/ЗУМ | «Слишком много данных» | Персональный стартовый экран по роли |
| УМ/ЗУМ | «Нет времени открывать» | Мобильный доступ, 1–3 мин утром |
| УМ/ЗУМ | «Цифры не совпадают с реальностью» | Прозрачность источников, метка качества |
| УМ | «Меня сравнивают несправедливо» | Только с релевантной группой по МКФ |
| Аналитик | «Нет нужного фильтра/выгрузки» | Расширенный режим |
| Дивизион | «Нужна общая картина, не детализация» | Отдельный режим витрины |

## Системный риск: «ещё один отчёт»
Случится, если: нет владельца, данные обновляются редко, нет регулярного использования.
**Противоядие:** первый релиз = только управленческий минимум:
Пульс РРС → карточка филиала → список действий → контроль эффекта.
