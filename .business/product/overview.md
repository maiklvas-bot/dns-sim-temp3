# Продукт: как устроен

## Суть
Live-симуляция рабочего дня директора магазина под наблюдением асессора. Участник реагирует на поток ситуаций и принимает управленческие решения в реальном времени.

## Поток оценки
```
Асессор запускает сессию → участник входит → идут сигналы по каналам
→ участник выбирает решения (варианты с разным уровнем)
→ решения влияют на метрики магазина и копят баллы по компетенциям
→ сессия завершается → расчёт профиля → результат + PDF-отчёт
```

## Каналы сигналов
| Канал | Что это |
|-------|---------|
| `main_case` | Основной кейс — ядро ситуации |
| `email` | Письма (почтовый ящик) |
| `messenger` | Сообщения в мессенджере |
| `video` | Видеозвонки/видеообращения |
| Сигналы зала | Зоны: торговый зал, склад, выдача, начальство |

## Сложность сессии
Три уровня — `easy` / `medium` / `hard` — различаются числом авто-кейсов, временем и временным коэффициентом оценки (см. [../methodology/scoring.md](../methodology/scoring.md)).

## Метрики магазина (в симуляции)
Решения участника влияют на: очередь (queue), конверсию (conversion), настрой команды (morale), влияние на выручку (revenue_impact), статус выдачи (delivery_status). Это создаёт правдоподобную обратную связь, но **итоговая оценка идёт по компетенциям**, а не по метрикам.
