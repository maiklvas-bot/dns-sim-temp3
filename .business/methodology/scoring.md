# Расчёт оценки

Как решения участника превращаются в профиль компетенций.

> Источник истины — `shared/simulation-scoring.ts`. Этот файл объясняет логику для бизнес-задач.

## Базовые принципы
- Каждый **вариант решения** (`option`) имеет `score` **0–5** и набор `competency_scores` (вклад в компетенции).
- Целевой уровень компетенции — **4.0 из 5** (на странице результатов сравнение идёт с этой планкой).
- Все средние округляются до десятых и ограничиваются диапазоном 0–5.

## Формула вклада в компетенцию
Для каждого выбранного решения вклад компетенции считается с двумя множителями:

```
вклад = competency_score × weightRatio × qualityRatio
```

- **weightRatio** — вес кейса. Для основных кейсов берётся из настроек `caseWeights` (в процентах, нормируется к 0–1). Для прочих = 1.
- **qualityRatio** — качество решения: `score / 5`, ограничено диапазоном **0.1–1** (даже слабое решение даёт минимум 10%).

Средний балл компетенции = сумма вкладов ÷ сумма весов (count).

## Временной коэффициент
Зависит от сложности сессии и применяется, **только если включён** `timeInfluenceEnabled`:

| Сложность | Коэффициент |
|-----------|-------------|
| easy | 0.95 |
| medium | 1.00 |
| hard | 1.08 |

Применяется к финальным средним (с ограничением 0–5). Идея: на высокой сложности корректные решения ценятся выше.

## Итог сессии
- `totalScore` — взвешенная сумма баллов решений × временной коэффициент.
- `averageScore` — средний балл по решениям (0–5).
- `competencyAverages` — средние по каждой компетенции → радар-профиль.

## Что видит оценщик
Hero-вердикт, радар компетенций с линией цели 4.0, сравнение «факт vs ожидание», план развития по компетенциям ниже цели. См. [../product/modules.md](../product/modules.md).
