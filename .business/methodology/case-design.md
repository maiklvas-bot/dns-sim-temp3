# Дизайн кейсов

Как устроен контент симуляции. Опора при создании/редактировании кейсов.

> Источник истины — типы в `shared/simulation-content.ts` и контент в `script/bootstrap-content.json`.

## Структура кейса (`SimCase`)
```
SimCase
├── title, description
├── primaryCompetencies[]      ← основные оцениваемые компетенции
├── secondaryCompetencies[]    ← второстепенные
├── trigger                    ← как кейс приходит (message/email/call/visitor/zone_signal)
├── zones_affected[]           ← торговый_зал / склад / выдача / начальство
└── cycles[]                   ← один или несколько шагов
```

## Цикл (`CaseCycle`)
Кейс — многошаговый. Каждый цикл:
- `situation` — описание ситуации на этот момент;
- `signal` — что видит/слышит участник (сообщение, сигнал зоны, письмо, звонок, визит);
- `options[]` — варианты решений;
- `priority` / `criticality` — нормальный / внимание / риск;
- может вести к следующему циклу (`nextCycleId`) — ветвление.

## Вариант решения (`CaseOption`)
| Поле | Смысл |
|------|-------|
| `level` | Уровень качества варианта (1 = слабый, выше = лучше) |
| `text` | Текст решения, который выбирает участник |
| `score` | Балл варианта 0–5 |
| `competency_scores` | Вклад в конкретные компетенции |
| `effects` | Влияние на метрики: queue, conversion, morale, revenue_impact, delivery_status |
| `nextChannel` / `nextCycleId` | Куда ведёт решение дальше |

## Принципы хорошего кейса
1. **Реалистичность** — ситуации из реальной работы директора магазина DNS (расстановка смены, конфликт, давление сверху, дефицит, жалоба клиента).
2. **Несколько компетенций** — один кейс оценивает 2–4 компетенции (primary + secondary).
3. **Градиент решений** — варианты от слабого к сильному, с разным `score` и `competency_scores`.
4. **Правдоподобные последствия** — `effects` отражают реальное влияние на магазин.
5. **Давление времени** — таймеры (`timing`) создают условия неопределённости.

## ⚠️ Правило изменения контента
Контент симуляции (кейсы, scoring, медиа) — **защищённый**. Не менять без явной задачи (см. `docs/TASK_RULES.md`). Контент — источник истины для оценки людей.
