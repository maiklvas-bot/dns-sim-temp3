# DNS SimCenter — второй мозг

> Входная точка при бизнес-задачах. Читай только нужный файл. При незнакомом термине DNS/РРС — `GLOSSARY.md` первым.

**Продукт:** DNS SimCenter — внутренний инструмент DNS для оценки управленческих компетенций кандидатов и руководителей розничных магазинов.
**Главная цель:** помощь в принятии более правильных кадровых решений.

## Структура

| Папка | О чём |
|-------|-------|
| `context/` | Зачем и где: про SimCenter, боль статус-кво, стейкхолдеры |
| `product/` | Что это: суть, экраны, направления развития |
| `methodology/` | ★ Ядро: модель компетенций, scoring, дизайн кейсов |
| `audience/` | Кто: роли системы, кого оцениваем |
| `strategy/` | Куда: цели, метрики успеха, масштабирование |
| `assets/` | Бренд: цвета, шрифты |
| `GLOSSARY.md` | Термины ассессмента + термины розницы DNS/РРС |

## Важно
- `.business/` — вне git.
- Контент симуляции (кейсы, scoring, медиа) — защищённый, не менять без явной задачи (`docs/TASK_RULES.md`).
- Источник истины по факту — код (`shared/`, `client/src/data/`, `script/bootstrap-content.json`); `.business/` хранит **смысл и бизнес-контекст**.
