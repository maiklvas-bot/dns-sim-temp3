# ЗРД UI — атомарный дизайн (Atomic Design)

Компоненты экрана игрока организованы по уровням Atomic Design. Новые компоненты добавляй
в соответствующий слой; организм собирается из молекул и атомов, а не дублирует разметку.

## Слои

### `atoms/` — атомы
Мельчайшие переиспользуемые элементы без бизнес-логики и состояния.
- `Chip` — компактный чип (эффект карты, стоимость, статус), тоны `pos | neg | neutral`.

### `molecules/` — молекулы
Небольшие узлы из атомов, отражающие одну сущность.
- `ZrdProjectCard` — карта проекта (категория + стоимость + чипы эффектов из `Chip`).

### `organisms/` — организмы
Самостоятельные секции игрового стола, собранные из молекул и атомов.
- `ZrdResourceBar` — панель 5 ресурсов
- `ZrdMetricsTracks` — треки показателей (терраформинг)
- `ZrdBoard` — поле региона
- `ZrdPhaseBar` — фазовая шкала квартала
- `ZrdHand` — рука/оффер карт (рендерит `ZrdProjectCard`)
- `ZrdActionDock` — док стандартных действий
- `ZrdEventDialog` — модал события (варианты из `Chip`)
- `ZrdGoalDeclaration` — выбор стратегии
- `ZrdResults` — итог партии + радар компетенций
- `ZrdLobby` — лобби/быстрый старт

### Шаблон (template) и страница (page)
- Шаблон-композиция — `../ZrdGameWorkspace.tsx` (раскладка стола, оркестрация фаз).
- Страница-роут — `client/src/pages/zrd.tsx` (`/zrd`).

## Правила
1. Организм не дублирует разметку чипа/бейджа — использует атом.
2. Бизнес-логика хода — в `useZrdGame`/сервере, не в презентационных компонентах.
3. Стили — токены `--zrd-*` из `client/src/styles/zrd.css` (работают в обеих темах).
