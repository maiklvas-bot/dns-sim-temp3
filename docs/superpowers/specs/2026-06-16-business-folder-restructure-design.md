# Реструктуризация `.business/` под DNS SimCenter — дизайн

**Дата:** 2026-06-16
**Статус:** утверждён пользователем (подход A+C)

## Проблема

Папка `.business/` («второй мозг») описывает **другой продукт** — «Пульс РРС Тюмень» (управленческий дашборд метрик филиалов). Реальный код в репозитории — это **DNS SimCenter**: live-симуляция для оценки управленческих компетенций. Вся business-документация рассинхронизирована с продуктом.

## Решение

Полностью переписать `.business/` под DNS SimCenter. Гибрид подходов A (знакомый верхний уровень) и C (методология ассессмента в ядре).

## Что за продукт (зафиксировано в брифинге)

**DNS SimCenter** — внутренний инструмент DNS для оценки управленческих компетенций кандидатов и руководителей розничных магазинов.

**Бизнес-задачи (все четыре):**
- оценка готовности перед назначением (кадровый резерв);
- отбор при найме на руководящие позиции;
- обучение и развитие (тренажёр, план развития по слабым компетенциям);
- аттестация действующих руководителей.

**Модель:** внутренний инструмент DNS с прицелом на масштабирование на другие РРС/дивизионы. Некоммерческий — разделы economics/marketing/pricing не нужны.

**Ценность / боль статус-кво:**
- оценка субъективна (нет единого стандарта);
- нет единой модели компетенций (нет общего языка);
- долго и дорого (требует времени экспертов, не масштабируется);
- **главная цель: помощь в принятии более правильных управленческих (кадровых) решений.**

## Фактура из кода (опора, не выдумка)

- **14 компетенций** в 3 категориях: `leadership` (планирование, основы менеджмента, делегирование, ответственность, контроль, ориентация на результат), `advanced` (коммуникация, принятие решений, стрессоустойчивость, бизнес-процессы, гибкость, ИТ-инструменты), `basic` (правовые основы, знание продукта).
- **Каналы сигналов:** `main_case`, `email`, `messenger`, `video` + сигналы зала.
- **Зоны:** торговый_зал, склад, выдача, начальство.
- **Структура кейса:** `SimCase` → `cycles[]` (многошаговые) → `options[]`; у опции `score` (0–5), `competency_scores` (вклад в компетенции), `effects` (queue, conversion, morale, revenue_impact, delivery_status).
- **Scoring:** взвешивание по весу кейса (`caseWeights` %), коэффициент качества (score/5, clamp 0.1–1), временной коэффициент по сложности (easy 0.95 / medium 1.0 / hard 1.08, если включено), усреднение по компетенциям, округление до 0.1, clamp 0–5. Ожидаемый уровень компетенции — **4.0/5**.
- **Сложности:** easy/medium/hard с разным числом авто-кейсов и временем.
- **Бренд (`dns-theme.ts`):** оранжевый `#F04E23`, тёмная тема navy `#0F1923`/`#1A2634`; шрифты — DNS RotondaC (файлы в `client/public/fonts/`), в теме указан Inter как fallback.
- **Роли системы (3):** `admin` (контент), `evaluator` (Оценщик — запуск/мониторинг сессии через экран «Асессор» **и** просмотр результатов), `participant`/`student` (прохождение). Staff-роли в `schema.ts` — только `admin` и `evaluator`; «Асессор» — экран Оценщика, не отдельная роль.
