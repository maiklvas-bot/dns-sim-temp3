# TASK-032: безопасная обработка ошибок API

## Объем изменения

- добавлен единый серверный обработчик необработанных API-ошибок;
- внутренние ошибки возвращают безопасное сообщение, стабильный код и `requestId`;
- тот же `requestId` записывается в серверный журнал вместе с технической причиной;
- PDF, XLSX, создание и восстановление сессий больше не возвращают исключения, stderr и системные пути;
- неизвестные ошибки парсинга и библиотек со статусом `400` не раскрывают технические сообщения;
- ожидаемые ошибки загрузки медиа сохраняют полезный статус `400` и отдельный код.

## Формат внутренней ошибки

```json
{
  "message": "Не удалось сформировать PDF.",
  "code": "PDF_EXPORT_FAILED",
  "requestId": "xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx"
}
```

Технический текст исключения доступен только в серверном журнале по `requestId`.

## Защищенные данные

В рамках задачи не изменялись:

- `.env`, логины и пароли;
- `data.db` и runtime-хранилища;
- миграции и структура базы;
- Docker-файлы;
- контент, система оценивания и медиаресурсы;
- клиентский интерфейс.

## Проверки

```text
npm run check
npm run test
npm run build
node script/check-docker-safety.mjs
git diff --check
```

Дополнительно на изолированной копии базы проверены:

- реальный ответ `PDF_EXPORT_FAILED` без Python stderr и системного пути;
- реальный ответ `INVALID_JSON` без позиции и текста ошибки парсера;
- совпадение `requestId` в HTTP-заголовке, JSON и серверном журнале;
- административный вход и открытие всех шести разделов;
- отсутствие новых ошибок браузерной консоли.

## Найдено вне периметра задачи

Функциональная проверка выявила существующие проблемы самих экспортов:

- PDF в локальном ESM-режиме обращается к недоступному `__dirname`;
- схема XLSX принимает строки-объекты, а генератор ожидает массивы ячеек.

TASK-032 не исправляет логику экспорта, а гарантирует безопасный ответ при таком сбое.
