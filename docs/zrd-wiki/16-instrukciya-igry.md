# 16. Инструкция настольной игры «Институт ЗРД: Покорение новых территорий»

> **Первоисточник инструкции.** Экранная версия для всех ролей — страница `/#/zrd/manual`
> (ссылки: лобби игры, шапка борда, мастер оценщика). Администратор правит ЭТОТ файл и
> экранную версию `client/src/features/zrd/manual/ZrdManualPage.tsx`.

**Формат:** стратегическая настольная игра в цифре · 1–4 игрока · 60–120 минут · 12+

---

## 1. Об игре

Вы — заместитель регионального директора DNS. Ваш стол — одна из четырёх РРС Дивизиона Урал
(Екатеринбург, Челябинск, Тюмень, Пермь). За игровой год (4 квартала) нужно развить регион:
открывать точки, строить логистику, растить команду, переживать чёрных лебедей — и обойти
соперников. Пока вы играете, алгоритм наблюдает за решениями и собирает профиль из
**12 компетенций ЗРД**.

### Состав игры

| Компонент | Сколько | Зачем |
|---|---|---|
| Столы (РРС) | 4 | Регион каждого игрока: свои ресурсы, показатели, колоды, миссии |
| Ресурсы | 5 (финансы, люди, материалы, технологии, репутация) | Валюта решений |
| Показатели | 3 базовых + 6 KPI (0–100%) | Здоровье региона, из них считаются миссии и ТР |
| Колоды | 6 личных × 50 карт | Главные ходы; добор каждый месяц, без повторов |
| Чёрные лебеди | 14 | Редкие сильные удары (локальные/глобальные) |
| Миссии | каталог из 10 | Цели года с поквартальной лестницей |
| Календарь | 4 квартала × 3 месяца = 12 тактов | Такт = месяц; неделя — единица длительности |

## 2. Роли

- **Испытуемый (игрок)** — получает код от оценщика и вводит его на **главном экране платформы**
  («Космонавт» → поле кода): один и тот же вход работает и для симуляции магазина, и для матча
  ЗРД — система сама определяет, куда посадить. Запасные пути: прямая ссылка от оценщика или
  `/#/zrd`. Видит только своё (рука/колоды/сброс) + публичные KPI соперников.
- **Оценщик** — собирает партию в кабинете (карточка «ЗРД»): сценарий, сложность 1–5, режим
  победы, состав стола, миссии, лебеди, темп; раздаёт коды; наблюдает; может запустить лебедя
  вручную и ставить паузу; получает профили компетенций по финалу.
- **Маскот** — фигурка, которой участник ходит по карте территории; выбирается при рассадке
  по территориям (мастер оценщика, шаг «Состав стола»). Четыре характера по стилю игры:
  **Стратег** — вдолгую: тир-3 проекты, производство, холодный расчёт; **Промо-гений** —
  агрессивный охват: реклама и акции, рискует кассой ради доли рынка; **Диспетчер** — процессы
  и логистика, устойчивость к чёрным лебедям; **Капитан команды** — люди, сервис, переговоры
  в кризисах. Аватар маскота виден в шапке борда, фишка — на карте.
- **ИИ-управленец** — занимает свободные столы, уровень 1–5 (5 — почти оптимально; 1 — ошибки,
  прожиг ходов, слепота к лебедям).
- **Пустой стол** — РРС не участвует.
- **Администратор** — правит контент игры и эту инструкцию (раздел 10), ведёт учётки.

## 3. Цель игры и победа

Режим выбирает оценщик до старта:

- **А. По итогам года** — 12 тактов, побеждает наибольший **Торговый рейтинг** =
  продажи + сервис + охват + бонусы выполненных миссий.
- **Б. Гонка к цели** — выбрана ключевая миссия (корона 👑); первый, кто довёл её KPI до
  финальной цели, побеждает немедленно.

**Тай-брейк:** меньше потрачено ресурсов → меньше действий → ничья. При равном результате
выигрывает более эффективный управленец.

## 4. Схема партии

```
КВАРТАЛ 1          КВАРТАЛ 2          КВАРТАЛ 3          КВАРТАЛ 4
[М1][М2][М3*]      [М4][М5][М6*]      [М7][М8][М9*]      [М10][М11][М12*]
                                                          * = рубеж квартала
Каждый месяц (такт):
  добор карт → действия всех столов одновременно → бросок лебедя →
  прогресс проектов (−4 нед.) → месячный доход
Рубеж квартала дополнительно:
  производство ресурсов/показателей → пересмотр целей миссий → дилемма каждому столу
```

**Темп:** 2–15 минут реального времени на такт (таймер в шапке). Не успел — авто-пас,
дилемма закрывается бесплатным вариантом. Матч не зависает из-за одного игрока.

## 5. Карта хода: ваш месяц за 6 шагов

1. **Осмотритесь** — «Показатели региона» и «Миссия»: какой KPI отстаёт от цели квартала.
2. **Решите дилемму** — блок «Событие раунда» ждёт выбора? Без решения месяц не завершить.
   Выбирайте вариант под ситуацию (касса/команда/склад), а не «по красивому тексту».
3. **Проверьте лебедя** — блок пульсирует красным → риск бьёт по вам каждый месяц.
   Клик → реакция (есть бесплатная) → штраф с вашей РРС снят.
4. **Потратьте действия** (1–2 на месяц) — карта из колоды справа или стандартное действие
   слева. Карты с длительностью уходят в «Проекты» и срабатывают по завершении недель.
5. **Доберите знания** — «Анализировать» открывает товарную колоду и засчитывается
   как работа с данными (аналитика в профиле).
6. **Завершите месяц** — оранжевая кнопка в центре; дальше ждём остальных.

## 6. Интерфейс борда (схема)

```
┌──────────────────────────────────────────────────────────────────────┐
│ (1) Шапка: Выход · Инструкция · «Кв 2/4 · Мес 05/12» · таймер ⏱      │
├───────────────┬──────────────────────────────────────┬───────────────┤
│ (2) Показатели│ (7) Миссия │ (8) Событие │ (9) Лебедь │ (13)          │
│     региона   ├──────────────────────────────────────┤  6 личных     │
│ (3) Доступные │                                      │  колод:       │
│     действия  │   (10) ЦЕНТР: карта 4 РРС            │  Продвижение  │
│ (4) Действия  │        [Завершить месяц]             │  Сервис       │
│     (глаголы) │                                      │  Логистика    │
│ (5) Ресурсы   ├──────────────────────┬───────────────┤  Товар        │
│ (6) Проекты   │ (11) 4 РРС: KPI всех │ (12) Сброс ·  │  Сотрудники   │
│               │      столов (ваш ярче)│      Раздача │  Проекты      │
└───────────────┴──────────────────────┴───────────────┴───────────────┘
```

| № | Элемент | За что отвечает | Как делать выбор |
|---|---|---|---|
| 1 | Шапка | Календарь и таймер дедлайна (за 60 сек. краснеет) | Следить за временем |
| 2 | Показатели региона | Живые данные вашей РРС | Наведение — подсказка |
| 3 | Доступные действия | 5 стандартных ходов | Клик = действие + финансы |
| 4 | Действия (глаголы) | Планировать/Строить/Развивать/Управлять/Анализировать | Клик открывает колоду справа |
| 5 | Ресурсы | 5 ресурсов + доход | Просмотр |
| 6 | Проекты | Длительные карты, прогресс «3/6 НЕД» | Наблюдение |
| 7 | Миссия | Прогресс против целей квартала; 👑 = ключевая | Ориентир для выбора карт |
| 8 | Событие раунда | Квартальная дилемма | «Выбрать реакцию» → модал вариантов |
| 9 | Чёрный лебедь | Активный риск (влияние/недели/масштаб) | Клик → реакция; наведение — описание |
| 10 | Центр — карта территории | Остров вашей РРС из гексов (четверть 1/4 карты дивизиона): по клеткам ходит маскот, клетки светлеют с ростом охвата, из сыгранных карт появляются магазины/склады/заводы/машины | Клик по соседней (пунктирной) клетке — шаг маскота, как в «Героях»; кнопка «Завершить месяц» — внизу карты |
| 11 | 4 РРС | Публичные KPI соперников, кто ходит | Наведение — карточка стола |
| 12 | Сброс · Раздача | Оранжевая «Сброс» — разыгранные карты; синяя «Раздача» — карты, выданные на партию и ждущие добора | Сброс: клик — список (виден только вам). Раздача: клик — количество по направлениям, содержимое — тайна до добора |
| 13 | 6 колод | Личные стопки: в руке/в колоде | Стопка → веер → карта → «Разыграть» |

## 7. Карты, события, лебеди

**Стартовая рука:** 4–8 карт в первый месяц (чем выше сложность, тем меньше), дальше добор
2–3 карты/месяц. Разыгранное уходит в оранжевый «Сброс» (открывается, виден только вам),
ещё не розданное лежит в синей «Раздаче» (видно только количество по направлениям).

**Как читать карту:** название + масштаб («…· пилот» → «…· дивизион»)· цена (К/П/С/Т/Р) ·
эффекты (чипы; «+N/кв» — производство) · условие (тир 2–3, напр. «продажи ≥ 5») ·
длительность («проект 6 нед.» → панель «Проекты», эффект по завершении).
Тир-1 — быстрые латки, тир-2 — рабочая лошадка, тир-3 — дорогие проекты, выигрывающие
партию при запуске не позже середины года.

**Дилеммы** (рубеж квартала): варианты равной формы, «правильного по тексту» нет —
оценивается уместность к состоянию вашей РРС. Блокируют завершение месяца.

**Чёрные лебеди (14):** вероятность за такт — выкл 0% / редко 10% / стандарт 22% / шторм 40%.
Локальный бьёт по одной РРС, глобальный — по всем. Штраф тикает каждый месяц до реакции
(или конца длительности). Реакция снимает штраф только с вашей РРС. Оценщик может запустить
лебедя вручную. Пул: проверка органов, банкротство поставщика, кибератака, эпидемия,
валютный шок, забастовка перевозчиков, пожар склада, репутационный кризис, федеральный
конкурент, кадровый дефицит, сбой ИТ, пересмотр аренды, демпинг маркетплейсов, блокировка рекламы.

## 8. Сценарии

| Сценарий | Суть | Особенности |
|---|---|---|
| Покорение новых территорий | основной, сбалансированный рост | лебеди стандарт, победа по году |
| Антикризисный год | шторм: удержать регион | урезанный старт, лебеди шторм, миссии удержания |
| Гонка за лидером | скорость решает | режим «гонка», ключевая миссия, добор к продвижению |
| Операционная эффективность | максимум из малого | урезанный старт, добор к логистике/товару |

Типовые составы: соло (1 человек + 3 ИИ) · дуэль (2+2) · командный зачёт (3–4 человека) ·
трое + пустой стол · демонстрация (4 ИИ).

## 9. Компетенции ЗРД и профиль

Профиль собирает алгоритм по журналу решений — **оценщик баллы не ставит**. Оценивается
**уместность выбора к контексту РРС** в момент решения, поэтому «выучить ответы» нельзя.

| Компетенция | Игровые сигналы |
|---|---|
| Планирование и организация | длительные проекты в первой трети года, производство, ровный темп |
| Постановка цели | закрытые миссии, движение KPI без «пилы» |
| Принятие решений | уместность в дилеммах и на лебедях |
| Аналитическое мышление | «Анализировать» перед решениями, товарные карты |
| Гибкость / адаптивность | скорость и уместность реакции на лебедей |
| Коммуникация | переговорные опции конфликтов (усиливается в мультиплеере) |
| Ориентация на результат | прогресс KPI к финальным целям, эффективность |
| Мотивация и команда | карты «Сотрудники», реакции на кадровые события |
| Критическое мышление | избегание слабых вариантов, трейд-офф карты |
| Инициативность | активность, ранние реакции, тир-3 карты |
| Управление конфликтами | опции конфликтных событий, сервисные кейсы |
| Стратегическое видение | тир-3 проекты, 4+ направлений, баланс показателей |

Шкала 0–5. Мягкие компетенции в соло видны слабее — усиливаются в партиях людей.

## 10. Итоги: карта развития (как после основной симуляции)

По финалу формируются: **игроку** — таблица матча (места по ТР, миссии, победитель);
**оценщику** — профиль 12 компетенций каждого участника + KPI + эффективность.

Карта развития сотрудника:
1. **Сильные стороны** (4.0+) — на что опираться, что делегировать.
2. **Зоны роста** (<3.0) — с конкретными игровыми эпизодами.
3. **Рекомендации** — книга/приём (раздел 11) + рабочая практика.
4. **Контрольная точка** — повторная партия через 2–3 месяца в другом сценарии, сравнение профилей.
