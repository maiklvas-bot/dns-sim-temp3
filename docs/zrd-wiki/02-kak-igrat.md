# 02. Как играть: ход и кварталы

## Структура партии
- Партия = **4 квартала** (год региона). Один квартал ≈ 12–15 мин → партия ~1 час.
- Время можно **ускорять ×1–10** (для тренировки/демо).
- В начале партии игрок **декларирует цель** (стратегию) — см. [09. Стратегии](09-strategii.md).

## Фазы одного квартала
Каждый квартал проходит четыре фазы по порядку:

```
1) Исследование → 2) Действия → 3) Событие → 4) Производство → (следующий квартал | конец)
```

### 1. Исследование (research)
- Игроку раскладывается **4 карты проектов** из колоды.
- Можно **оставить** часть карт в руку: **первая оставленная — бесплатно, каждая следующая — 2 К**.
- Невзятые карты уходят в сброс. Оставленные доступны для розыгрыша в этом и будущих кварталах.

### 2. Действия (action)
- Доступно **2 действия за квартал** (на лёгких уровнях 1–2 — **3 действия**).
- Каждое действие — одно из:
  - **Разыграть карту** из руки (оплатить стоимость, выполнить условие) — [05. Карты](05-karty-proektov.md);
  - **Стандартное действие** (всегда доступно) — [06. Стандартные действия](06-standartnye-deystviya.md);
  - **Открыть данные (BI)** — бесплатный просмотр аналитики перед решением (не тратит действие);
  - **Пас** — завершить фазу действий досрочно.
- Когда действия закончились или нажат «Пас» → переход к событию.

### 3. Событие (event)
- Вскрывается **одно событие** (кризис/дилемма) с 2–3 вариантами реакции — [07. События](07-sobytiya.md).
- У события может быть **базовый урон**, если регион к нему не готов (напр. слабый склад).
- Сила негативных эффектов масштабируется **множителем наказания** от сложности.

### 4. Производство (production)
В конце квартала автоматически:
1. К запасам прибавляется **производство ресурсов** (в т.ч. капитал = доход).
2. К показателям прибавляется **производство показателей** (prodS/N/O) и временные модификаторы
   (напр. наставничество даёт +prodNPS на 2 квартала).
3. Проверка конца партии — [11. Условия победы](11-pobeda.md).

> ⏱ Важно по таймингу: производство падает **в конце** квартала. Поэтому в середине квартала
> вы тратите то, что накопили ранее — планируйте денежный поток.

## Поток данных (для разработки)
Клиент/бот шлёт **намерение хода** (`TurnIntent`), сервер-движок (`applyIntent`) проверяет фазу,
применяет эффект, пишет лог. Состояние **серверно-авторитетное** — накрутить с клиента нельзя.
Движок детерминированный (seeded RNG) → партия точно воспроизводится из лога для реплея.
