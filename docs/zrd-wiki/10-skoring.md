# 10. Скоринг компетенций (12)

Пока игрок развивает регион, движок **наблюдает за решениями** и формирует профиль из 12
компетенций ЗРД (баллы **ФАКТ 0–5**). Оценщик баллы **не проставляет** — он трактует готовый
результат. Полная методика и калибровка: `docs/zrd-scoring-v1.md`.

## Принцип §8a — анти-хак (главное)
Компетенция считается **по уместности выбора к контексту**, а не по тексту варианта.
- В момент решения фиксируется контекст состояния (мало/много капитала, есть персонал/технологии,
  отстаём от целей, движок стабилен…).
- Выбор сравнивается с тем, **когда он уместен**. Совпал с ситуацией → сильный сигнал; выбран
  заведомо слабый вариант → штраф.
- **Числа в вариантах при оценке роли не играют.** Нельзя «вычислить правильный ответ по форме»,
  как это бывало в прошлой симуляции (длинный/сложный = правильный). При этом любой разумный ход
  двигает игру вперёд — игрок не застревает.

## 12 компетенций и их «крючки»
| Компетенция | Что в игре её сигналит |
|---|---|
| Планирование и организация | ранний движок (доход/долгосрочные карты в Q1–Q2), построенный prodК, активность, дисциплина добора карт |
| Постановка цели | декларация стратегии + консистентность ходов с ней |
| Принятие решений | уместные выборы событий против слабых; действие вместо паралича |
| Аналитическое мышление | использование BI-данных перед ходом + IT-карты |
| Гибкость / адаптивность | адаптивная реакция на события по контексту |
| Коммуникация | совещание в конфликте точек (+ базовый уровень; в solo слабо наблюдаема) |
| Ориентация на результат | прогресс показателей к целям к концу партии |
| Мотивация и построение команды | HR-вложения + «обучить» в текучести кадров |
| Критическое мышление | избегание заведомо слабых вариантов |
| Инициативность | проактивность: число проектов, стратегические карты, декларация |
| Управление конфликтами | разрешение конфликта точек (совещание > волевое > игнор) |
| Стратегическое видение | долгосрочные карты, охват категорий, сбалансированность показателей |

## Шкала и калибровка
`балл = clamp(1 + (сырой_сигнал / EXPECTED_MAX) * 4, 0, 5)`. Пороги `EXPECTED_MAX` откалиброваны
эмпирически (30 партий, L3) так, что **компетентная средняя игра ≈ 3.0**, слабая — заметно ниже,
выдающаяся — выше. Профили персон (planner/balanced/weak…) и автопроверки — в `docs/zrd-scoring-v1.md`.

## Честные ограничения
- **Мягкие компетенции** (Коммуникация, Команда, Конфликты) в solo ловятся слабо — лишь через
  HR/конфликт-события; полноценно — в мультиплеере (шаг 2).
- Алгоритм уверенно отличает **слабую игру от компетентной**; тонкое «средний vs сильный» по
  событийным компетенциям ограничено до партий **живых людей** (валидация — Фаза 6).
