# Changelog

## 2026-07-02 — ЗРД-борд: блок «Чёрный лебедь», управленцы РРС, сценарии игрока

- Замыкающий блок «Чёрный лебедь» в верхнем ряду борда (риск-событие раунда: влияние/длительность/вероятность/масштаб); всплывающее окно по наведению с пояснением механики и текущего события; мягкая красная пульсация кромки, когда риск активен (не задевает соседние блоки, уважает `prefers-reduced-motion`).
- Управленец каждой РРС: в шапке блока — значок человека (живой игрок) или робота (ИИ) в цвет РРС (заменил тег «KPI региона»). Подробная карточка РРС по наведению: тип управленца (сценарий + компетенции / уровень ИИ), особенность региона, текущие показатели.
- 5 сценариев поведения игрока на компетентностном подходе (`zrd-player-scenarios.ts`): системный стратег, антикризисный менеджер, драйвер роста, операционный оптимизатор, наставник команды. Состав задаёт оценщик, сценарий выбирает игрок при входе (пока демо-заглушки до live-сессии).
- Служебное: `tmp/` и `.playwright-mcp/` добавлены в `.gitignore` (эфемерные артефакты); детальный журнал — `docs/zrd-wiki/14-changelog.md`.
- Проверка: `tsc` = 0, `build` ✓, Playwright-скрипты (1920×1080) — попапы/пульс, без ошибок в консоли. Только UI/клиент; логика/сервер не затронуты.

## 2026-06-16 — UI: масштабируемость, dvh, мобильные табы участника, фундамент токенов темы

- Масштаб: убран `maximum-scale=1` (зум разрешён), `viewport-fit=cover`, `<title>`/`lang=ru`/`theme-color`; корневой `font-size` сделан флюидным (`clamp`) — rem-система масштабируется под разрешение/зум 60–120%.
- Высоты `100vh`/`h-screen` → `dvh` (симуляция, оболочки, журнал решений, результаты) — корректная высота в мобильных браузерах.
- Мобайл: табы «Карта/Сигналы/Метрики» включены и для участника на телефоне (`isReadOnly || isMobile`); десктоп без изменений.
- Тема: добавлен светлый набор семантических токенов (`--background/--card/--foreground/--border/--muted-foreground/…`), `--primary` остаётся оранжевым — фундамент для полного перехода с хардкод-цветов на токены.
- Проверка: `tsc`, `build`, `test:browser` (темы/переполнение/консоль), скрины dark/light на 390/1440/2560 — без регрессий и горизонтального скролла. Только UI/frontend; логика/данные/server не затронуты.
